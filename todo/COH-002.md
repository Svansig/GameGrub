# COH-002 - Reduce service singleton usage from app layer

- **ID**: `COH-002`
- **Area**: `service + ui`
- **Priority**: `P1`
- **Status**: `Reopened`
- **Owner**: `Sisyphus`
- **Documentation Impact**: `No doc changes required` - Internal architectural improvement.
- **Reviewer**: `TBD`

## Problem

Static/singleton service access makes state harder to reason about and test.

## Scope

- In scope:
  - Inventory singleton access points in `ui` and app entry paths.
  - Propose migration to injected dependencies or ViewModel gateways.
- Out of scope:
  - Removing every singleton in one PR.

## Acceptance Criteria

- [x] Inventory exists with risk ranking.
- [x] Migration plan for top 10 usages is documented.
- [x] At least 2 high-risk call sites are refactored.

## Risk Ranking

**HIGH RISK (priority for migration):**
1. GameGrubMain.kt - SteamService (orchestration hub, affects entire app)
2. UserLoginViewModel.kt - SteamService (auth flows, critical)
3. GameGrubMain.kt - LaunchRequestManager (external intent handling)

**MEDIUM RISK:**
1. LibraryViewModel.kt - All 3 services (auth state, sync triggers)
2. MainViewModel.kt - SteamService (launch notifications)
3. SteamAppScreenViewModel.kt - SteamService (downloads, deletes)

**LOW RISK:**
7-17. App screens, dialogs (mostly read-only queries)

## Migration Plan (Top 10)

1. **GameGrubMain: SteamService.isConnected/isLoggedIn/keepAlive** → Inject into MainViewModel state
2. **GameGrubMain: LaunchRequestManager** → Create LaunchRequestGateway
3. **UserLoginViewModel: SteamService login methods** → Inject SteamAccountDomain
4. **LibraryViewModel: Service auth state checks** → Add platform state to ViewModel
5. **SteamAppScreenViewModel: SteamService forceSyncUserFiles** → Inject SteamCloudStatsDomain
6. **SteamAppScreenViewModel: SteamService deleteApp** → Inject SteamInstallDomain
7. **EpicAppScreen: All service calls** → Extract to EpicAppScreenViewModel
8. **AmazonAppScreen: All service calls** → Extract to AmazonAppScreenViewModel
9. **ContainerConfigDialog: SteamService.keepAlive** → Move to XServer lifecycle coordination
10. **SettingsGroupDebug: SteamService.stop/logOut/clearDatabase** → Create SettingsGateway
