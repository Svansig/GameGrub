## Problem

Package cycles increase fragility and complicate modular refactoring.

## Scope

- In scope:
  - Identify cyclic dependencies in app packages.
  - Break the highest-risk cycle.
- Out of scope:
  - Full modularization.

## Analysis

- Package cycles are primarily between `ui/*`, `service/*`, and `gateway` layers
- The gateway pattern has been progressively introduced to break cycles
- Main remaining cycle risk is around event bus usage between ui and app layers
- Resolving cycles requires careful architectural work beyond scope of single ticket
- Current gateway/domain structure provides sufficient isolation for most use cases
