# COH-017 - Unify orientation preference flow behind a single manager-owned policy contract

- **Reviewer**: `TBD`

## Problem

Orientation policy currently comes from multiple sources (prefs, route logic, container overrides) without a single contract, increasing coupling and ambiguity.

## Scope

- In scope:
  - Define one policy contract for orientation inputs: user preference, per-session override, and fallback.
  - Ensure writes to `requestedOrientation` only occur through `OrientationManager`.
  - Align `PrefManager` and settings dialog behavior with manager-owned policy flow.
- Out of scope:
  - Full migration to a new preferences backend.
  - New orientation UI features.
