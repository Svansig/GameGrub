# COH-023 - Consolidate network infrastructure under `NetworkManager`

- **ID**: `COH-023`
- **Area**: `network + utils/network + service/api consumers`
- **Priority**: `P1`
- **Status**: `Done`

- **Reviewer**: `TBD`

## Problem

Network concerns were split across `NetworkMonitor` and `utils/network/Net`, creating duplicated ownership for connectivity state and HTTP infrastructure.

## Plan

- [x] Move process-wide connectivity state ownership into `app.gamegrub.network.NetworkManager`.
- [x] Remove `NetworkMonitor` after migration completion.
- [x] Remove `utils/network/Net` compatibility wrapper after migration completion.
- [x] Migrate representative app/service/api call sites from `Net.http` to `NetworkManager.http`.
- [x] Update architecture docs for the new boundary and migration posture.

## Scope

- In scope:
  - Centralized connectivity state and init flow in `NetworkManager`.
  - Network client/call helper ownership under `NetworkManager`.
  - Migration of direct `Net.http` usage in touched platform/api files.
-  - Removal of legacy wrapper surfaces once no callers remain.
- Out of scope:
  - Business API behavior changes.
  - Transport-policy changes unrelated to ownership consolidation.

## Acceptance Criteria

- [x] `NetworkManager` owns reactive connectivity state and initialization.
- [x] `NetworkMonitor` is removed and all call sites use `NetworkManager` directly.
- [x] `utils/network/Net` compatibility wrapper is removed.
- [x] No direct `Net.http` usage remains in main source set.
- [x] Architecture documentation reflects the ownership boundary.

## Validation

- [x] Static search: `Net.http` usage removed from `app/src/main/java`.
- [x] Static search: `NetworkMonitor` symbol removed from production Kotlin sources.
- [x] File diagnostics run for changed network-related files.
- [ ] `./gradlew :app:compileDebugKotlin` (blocked locally if `JAVA_HOME` is not configured).
