# COH-025 - Move auth and integrity utilities to explicit auth/security boundaries

- **ID**: `COH-025`
- **Area**: `utils/auth + auth/security`
- **Priority**: `P1`
- **Status**: `Done`
- **Owner**: `TBD`

- **Reviewer**: `TBD`

## Problem

`utils/auth` contains workflow owners (OAuth orchestration, key attestation, Play Integrity lifecycle, platform sign-in state aggregation), not generic helpers.

## Scope

- In scope:
  - Move `AuthUrlRedaction`, `KeyAttestationHelper`, `PlatformAuthUtils`, `PlatformOAuthHandlers`, and `PlayIntegrity` to owned auth/security packages.
  - Replace static/singleton call paths with injectable coordinators/gateways where appropriate.
  - Update call sites in app/service/UI layers.
- Out of scope:
  - Changing auth provider API behavior.

## Changes Made

Moved all 5 files from `utils/auth` to `service/auth`:
- `AuthUrlRedaction.kt` -> `service/auth/AuthUrlRedaction.kt`
- `PlayIntegrity.kt` -> `service/auth/PlayIntegrity.kt`
- `KeyAttestationHelper.kt` -> `service/auth/KeyAttestationHelper.kt`
- `PlatformAuthUtils.kt` -> `service/auth/PlatformAuthUtils.kt`
- `PlatformOAuthHandlers.kt` -> `service/auth/PlatformOAuthHandlers.kt`

Updated imports in call sites:
- `GOGOAuthActivity.kt`, `EpicOAuthActivity.kt`, `AuthWebViewDialog.kt` - use new `redactUrlForLogging` import
- `GameNativeApi.kt` - uses new `PlayIntegrity` import
- `UserLoginScreen.kt`, `PlatformOAuthHandler.kt` - use new `PlatformOAuthHandlers` import
