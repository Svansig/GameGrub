# COH-026 - Move container launch/preinstall ownership out of utils

- **ID**: `COH-026`
- **Area**: `utils/container + utils/preInstallSteps + utils/launchdependencies + launch`
- **Priority**: `P1`
- **Status**: `Backlog`

- **Reviewer**: `TBD`

## Problem

Container creation/config mutation, launch dependency checks, pre-install installer sequencing, and external intent launch override state are implemented under `utils`, but these are core launch/container workflow owners.

## Scope

- In scope:
  - Move `ContainerUtils` and `ContainerMigrator` into explicit container managers/gateways.
  - Move `LaunchDependencies` + `launchdependencies/*` to launch dependency domain.
  - Move `PreInstallSteps` + `preInstallSteps/*` to launch preinstall domain.
  - Move `IntentLaunchManager` and temporary override state to launch request/coordinator ownership.
- Out of scope:
  - Behavioral changes to launch result semantics.

## Decomposition Plan (Current Pass)

Primary hotspot: `app/src/main/java/app/gamegrub/utils/container/ContainerUtils.kt`

Planned split boundaries:
- Container creation/default configuration -> container manager/domain owner.
- Executable discovery and launch-oriented heuristics -> launch/install domain owner.
- Storage/path and marker operations -> storage gateway ownership.
- Game-source specific lookup glue -> platform/domain-specific owners.

### Implementation Checklist

- [ ] Inventory `ContainerUtils` functions by owner boundary (container, launch/install, storage, platform lookup).
- [ ] Move container lifecycle functions first and keep old entry points as temporary wrappers only where required.
- [ ] Move executable/launch helper logic to launch-owned package and update callers.
- [ ] Remove direct storage/file ownership from utils in favor of storage gateways.
- [ ] Delete obsolete wrappers/imports after call-site migration and confirm parity via launch smoke checks.
