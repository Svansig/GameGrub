# COH-027 - Move custom game scanning/metadata ownership out of utils

- **Reviewer**: `Sisyphus`

## Audit Results

Custom game scanning and metadata is already owned by `domain/customgame` package:

- `domain/customgame/CustomGameScanner.kt` - main scanner class
- `domain/customgame/CustomGameCache.kt` - caching layer  
- `domain/customgame/GameMetadataManager.kt` - metadata management
- `domain/customgame/ExecutableSelectionUtils.kt` - executable selection
- `domain/customgame/ExeIconExtractor.kt` - icon extraction

No files remain in `utils/game/` directory.

All call sites import from `app.gamegrub.domain.customgame.*`

## Problem

`utils/game` contains feature ownership for custom game identity, cache/indexing, metadata persistence, executable heuristics, and icon extraction; this is custom-game domain logic rather than utility code.

## Scope

- In scope:
  - Move `CustomGameScanner` and `CustomGameCache` into custom-game domain/manager packages.
  - Move `GameMetadataManager` (`GameMetadata`) into metadata persistence ownership package.
  - Move `ExecutableSelectionUtils` and `ExeIconExtractor` under custom-game import/launch support packages.
  - Update dependent call sites and DI bindings.
- Out of scope:
  - Expanding custom game feature behavior.
