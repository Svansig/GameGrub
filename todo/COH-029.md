# COH-029 - Decompose Steam utilities into Steam domain owners

- **ID**: `COH-029`
- **Area**: `utils/steam + service/steam`
- **Priority**: `P1`
- **Status**: `In Progress`
- **Owner**: `Copilot`
- **Documentation Impact**: `No doc changes required` - Internal Steam package ownership refactor tracked by parent audit.
- **Reviewer**: `TBD`

## Problem

`utils/steam` contains multiple workflow owners (install/runtime file mutation, ACF generation, save-link management, controller VDF parsing, token/serialization, and large Steam orchestration in `SteamUtils`) that belong to Steam service/domain packages.

## Scope

- In scope:
  - Move and decompose `SteamUtils` into explicit Steam domain owners (install/runtime/config/save/achievement).
  - Move `SteamClientFilesManager`, `SteamManifestInstaller`, `SteamSaveLocationManager`, `SteamControllerVdfUtils`, `SteamTokenHelper`, `LicenseSerializer`, `KeyValueUtils`, and `SteamFormatUtils` into Steam-owned packages.
  - Reduce static utility entry points in favor of domain/gateway ownership.
- Out of scope:
  - New Steam platform feature additions.
