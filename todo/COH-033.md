## Problem

`GameGrubApp` companion object retains mutable global references to runtime objects and UI views (`xServerView`, `inputControlsView`, `touchpadView`, listener refs), creating memory leak and thread-safety risk.

## Scope

- In scope:
  - Inventory and classify companion globals by ownership and lifecycle.
  - Replace long-lived view/listener references with lifecycle-owned holders and explicit cleanup points.
  - Introduce thread-safe access patterns for remaining shared runtime session state.
- Out of scope:
  - Redesigning Winlator runtime behavior.
