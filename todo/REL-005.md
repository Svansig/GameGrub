## Problem

Timeout behavior varies by call path, causing inconsistent user experience and recovery.

## Scope

- In scope:
  - Define timeout classes by operation type.
  - Apply policy to critical auth/library/download flows.
- Out of scope:
  - Network stack replacement.

## Validation

- [x] Timeout policy already defined and standardized:
  - NetworkUtils: 30s connect, 60s read, 30s write, 5min call
  - GOG/Epic/Amazon APIs: 30s connect, 30-120s read
  - UpdateChecker: 10s connect/read
  - KeyAttestationHelper: 10s connect/read
  - ManifestInstaller: 240s (4 min) timeout for installs
  - DownloadInfo: 5s await completion timeout
