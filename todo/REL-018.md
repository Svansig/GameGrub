## Problem

Startup migration work launched from `GameGrubApp` can fail without a clear policy for exception handling, retry, and failure telemetry, which risks silent startup instability.

## Scope

- In scope:
  - Define explicit failure policy for startup migration work (log, retry/defer, and safe fallback).
  - Ensure coroutine exception handling is deterministic and cancellation-safe.
  - Add focused tests for migration failure and fallback behavior.
- Out of scope:
  - Rewriting container migration implementation details.
