## Problem

The final APK still ships prebuilt arm64 libraries with 4 KB alignment for the PulseAudio chain:
`libltdl.so`, `libpulse.so`, `libpulseaudio.so`, `libpulsecommon-13.0.so`, `libpulsecore-13.0.so`, and `libsndfile.so`.
These are runtime-required and currently have no in-repo source producer wired into Gradle/CMake.

## Scope

- In scope:
  - Identify and vendor reproducible source producers for the PulseAudio/ltdl/libsndfile stack.
  - Add native build wiring (CMake and `externalNativeBuild`) for arm64 outputs.
  - Replace packaged prebuilt arm64 artifacts with source-built outputs.
  - Verify final APK 16 KB alignment via `:app:verifyDebug16KbPageSize`.
- Out of scope:
  - Audio driver behavior changes.
  - UI changes unrelated to native producer ownership.

### Resolved: libsndfile and libltdl (2026-04-10)
- `libsndfile.so` now source-built via `third_party/libsndfile/CMakeLists.txt` (FetchContent, libsndfile 1.2.2).
  - SHA-256 pinned: `3799ca9924d3125038880367bf1468e53a1b7e3686a934f098b7e1d286cdb80e`

- `libltdl.so` now source-built via `third_party/libltdl/CMakeLists.txt` (in-repo minimal shim).
  - `libpulsecore-13.0.so` only uses 5 lt_dl* symbols (confirmed via `strings`).
  - Shim: `third_party/libltdl/libltdl_android.c` — implements 5 functions over bionic dlopen.
  - CMake target `ltdl`; prebuilt deleted from jniLibs; Gradle exclusion added.
  - Verified: `LOAD align 2**14` in final APK.

### Resolved: PulseAudio stack (2026-04-10 — Phase 3)
- Official PA 13.0 cross-compiled for Android arm64 using Meson + NDK r28.
- Android compatibility patches applied:
  - `atomic.h`: cast `void *` to `(unsigned long)(uintptr_t)p` in `pa_atomic_ptr_store`
  - `config.h`: unset `HAVE_CPUID_H` (x86-only) and `HAVE_SYS_CAPABILITY_H` (not in bionic)
  - `src/map-file`: removed `pa_glib_*` and `pa_simple_*` from version script (in separate .so files)
  - `src/daemon/ltdl-bind-now.c`: `#undef PA_BIND_NOW` for Android (vtable API not in our ltdl shim)
  - NDK sysroot stubs added: `libintl.h` (no-op gettext), `ltdl.h` (minimal types + declarations)
  - `libltdl.so` shim expanded: `lt_dlinit`, `lt_dlexit`, `lt_dlsetsearchpath`, `lt_dlforeachfile`, `lt_preloaded_symbols`
  - Cross-file: `--undefined-version` linker flag for libpulse-simple.so version script
- All 4 outputs in `app/src/main/jniLibs/arm64-v8a/` verified at LOAD align 0x4000.
- Staged outputs also in `prebuilt-out/android/arm64-v8a/lib/`.
