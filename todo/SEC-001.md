## Audit Results

### Storage Locations

| Platform | File Location | Encryption |
|----------|---------------|------------|
| GOG | `context.filesDir/gog_auth.json` | None (plain JSON) |
| Epic | `context.filesDir/epic/credentials.json` | None (plain JSON) |
| Amazon | `context.filesDir/amazon_credentials.json` | None (plain JSON) |
| Steam | Container-managed via Steam binaries | Steam binary encryption |

### Contents (Plain Text)

- **GOG**: `access_token`, `refresh_token`, `user_id`
- **Epic**: `access_token`, `refresh_token`, `account_id`, `display_name`, `expires_at`
- **Amazon**: `accessToken`, `refreshToken`, `userId` (verified)

### Security Assessment

- All tokens stored in app-private internal storage (`filesDir`)
- Not accessible to other apps without root
- **Gap**: Not using Android's `EncryptedSharedPreferences` or equivalent
- Tokens are short-lived OAuth tokens, limiting exposure window
- Logout properly clears credential files

### Recommendations (Deferred)

1. Migrate to EncryptedSharedPreferences for credential storage
2. Add token encryption layer before storage
3. Document token lifecycle and refresh behavior
