## Problem

Dependency vulnerability checks are not codified into regular maintenance workflow.

## Scope

- In scope:
  - Define cadence and ownership for CVE reviews.
  - Add a lightweight dependency risk report process.
- Out of scope:
  - Immediate upgrade of all dependencies.

## Acceptance Criteria

- [ ] CVE review process doc added.
- [ ] Initial dependency risk baseline recorded.
- [ ] Follow-up upgrade tickets created for critical findings.

## Validation
