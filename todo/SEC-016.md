# SEC-016 - Gate analytics SDK startup by privacy consent in app initialization

- **Area**: `app startup + analytics/privacy`

- **Status**: `Backlog`
- **Owner**: `TBD`

- **Reviewer**: `TBD`

## Problem

Analytics SDK startup currently lacks an explicit app-start consent gate in initialization flow, creating risk that analytics identity/profile behavior runs before user privacy preferences are applied.

## Scope

- In scope:
  - Define and implement consent-aware analytics initialization flow.
  - Ensure analytics profile/identification behavior aligns with configured privacy state.
  - Add tests that verify analytics is disabled or delayed when consent is not granted.
- Out of scope:
  - Replacing analytics provider.
