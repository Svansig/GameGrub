## Audit Findings

PostHog initializes unconditionally in `GameGrubApp.onCreate()` at line 96-104:

```kotlin
val postHogConfig = PostHogAndroidConfig(
    apiKey = BuildConfig.POSTHOG_API_KEY,
    host = BuildConfig.POSTHOG_HOST,
).apply {
    personProfiles = PersonProfiles.ALWAYS
}
PostHogAndroid.setup(this, postHogConfig)
```

### Current Behavior

- PostHog initializes immediately on app start
- No consent check before initialization
- No user-settable privacy preference exists in PrefManager
- All events capture immediately regardless of privacy preference

### Security Gap

- **No consent gate**: Analytics starts before any user privacy choice
- **No opt-out mechanism**: No preference toggle in settings
- PostHog uses `personProfiles = PersonProfiles.ALWAYS` which identifies all events
