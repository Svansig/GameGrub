# SRV-001 - Move download orchestration to SteamInstallDomain

- **ID**: `SRV-001`
- **Area**: `service/steam`
- **Priority**: `P1`
- **Status**: `Backlog` (High priority but complex - requires careful orchestration of domain migration)
- **Owner**: `TBD`

## Problem

Download orchestration logic (~300 lines) is in SteamService companion object. This should be in SteamInstallDomain following the ownership matrix guidelines: domains own business workflows, service should only handle Android lifecycle.

## Scope

- In scope:
  - Move `downloadApp(appId: Int)` entry point to domain
  - Move `downloadApp(appId, dlcAppIds, isUpdateOrVerify)` overload to domain
  - Move main `downloadApp(...)` with DepotDownloader orchestration to domain
  - Move `completeAppDownload(...)` method to domain
  - Move `AppDownloadListener` inner class to domain package
  - Handle coroutine scope injection/parameter passing
- Out of scope:
  - UI layer changes
  - Test changes

## Acceptance Criteria

- [ ] All three `downloadApp` overloads removed from SteamService companion
- [ ] `completeAppDownload` removed from SteamService companion
- [ ] `AppDownloadListener` moved to domain package
- [ ] Download works end-to-end after migration

## Validation

- [ ] Manual test: download a game and verify it completes
- [ ] `./gradlew compileDebugKotlin` passes for touched files
