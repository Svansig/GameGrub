# SRV-004 - Move session/launch orchestration to SteamSessionDomain

- **ID**: `SRV-004`
- **Area**: `service/steam`
- **Priority**: `P1`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

Session and launch orchestration methods in SteamService companion should be in SteamSessionDomain:
- `beginLaunchApp` - already delegates to sessionDomain but wrapper remains
- `forceSyncUserFiles` - already delegates to cloudStatsDomain
- `closeApp` - already delegates to sessionDomain
- `notifyRunningProcesses` - mostly delegates

These wrappers should be removed, call sites should use domain directly.

## Scope

- In scope:
  - Remove wrapper methods that already delegate to domains
  - Update call sites to use domain methods directly
- Out of scope:
  - Any API changes to external callers

## Acceptance Criteria

- [ ] Wrapper methods removed from SteamService companion
- [ ] Call sites updated to use domain methods
- [ ] Launch/close flows still work

## Validation

- [ ] Manual test: launch a game
- [ ] Manual test: close a game
