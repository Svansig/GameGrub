# SRV-005 - Move login helper to SteamAccountDomain

- **ID**: `SRV-005`
- **Area**: `service/steam`
- **Priority**: `P2`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

`login(...)` private method in SteamService companion creates LogOnDetails and calls steamUser.logOn. This is authentication orchestration that belongs in account domain or auth service. The public login methods (`startLoginWithCredentials`, `startLoginWithQr`, `completeLoginWithAuthTokens`) already delegate to accountDomain.authService, but the internal `login` method still lives in companion.

## Scope

- In scope:
  - Move `login(...)` method logic to account domain or auth service
  - Keep public API facades if needed for compatibility
- Out of scope:
  - Changes to auth service implementation

## Acceptance Criteria

- [ ] `login` method removed from SteamService companion
- [ ] Authentication still works (QR and credentials)

## Validation

- [ ] Manual test: login with QR
- [ ] Manual test: login with credentials
