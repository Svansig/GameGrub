# SRV-006 - Move PICS/sync utility methods to appropriate domains

- **ID**: `SRV-006`
- **Area**: `service/steam`
- **Priority**: `P2`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

PICS-related utility methods in SteamService companion should be in appropriate domains:
- `checkWifiOrNotify()` - network policy, could stay as helper
- `hasActiveOperations()` - checks downloadJobs (domain) + session operations (domain)
- `isUpdatePending(...)` - PICS logic, belongs in PicsSyncDomain or libraryDomain
- `checkDlcOwnershipViaPICSBatch(...)` - already uses catalogManager, belongs in libraryDomain

## Scope

- In scope:
  - Move PICS utility methods to appropriate domains
  - Keep simple checkWifiOrNotify as private helper if needed
- Out of scope:
  - Any PICS sync business logic changes

## Acceptance Criteria

- [ ] Methods moved to domains
- [ ] Existing callers work unchanged
