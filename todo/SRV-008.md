# SRV-008 - Remove library wrapper facades from companion

- **ID**: `SRV-008`
- **Area**: `service/steam`
- **Priority**: `P3`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

SteamService companion has many wrapper methods that just delegate to libraryDomain:
- `getLicensesFromDb()`
- `getPkgInfoOf()`, `getAppInfoOf()`, `getDownloadingAppInfoOf()`
- `getDownloadableDlcAppsOf()`, `getHiddenDlcAppsOf()`
- `getInstalledApp()`, `getInstalledDepotsOf()`, `getInstalledDlcDepotsOf()`
- `getAppDownloadInfo()`
- `getAppDlc()`, `getOwnedAppDlc()`
- `getMainAppDlcIdsWithoutProperDepotDlcIds()`
- `refreshOwnedGamesFromServer()`
- `hasPartialDownload()`
- `filterForDownloadableDepots()`
- `getMainAppDepots()`, `getDownloadableDepots()` (both overloads)
- `getAppDirName()`, `resolveExistingAppDir()`, `getAppDirPath()`

These should be removed after call sites migrate to use libraryDomain directly.

## Scope

- In scope:
  - Track where each wrapper is used
  - Update call sites to use domain directly
  - Remove wrapper methods
- Out of scope:
  - Breaking changes without migration plan

## Acceptance Criteria

- [ ] Wrapper methods removed from companion
- [ ] Call sites updated
