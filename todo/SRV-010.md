# SRV-010 - Move Steam Input config resolution to SteamInstallDomain

- **ID**: `SRV-010`
- **Area**: `service/steam`
- **Priority**: `P2`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

Steam Input configuration methods in SteamService companion already delegate to installDomain but are wrapped unnecessarily:
- `resolveSteamInputManifestFile(...)` - already calls installDomain
- `loadConfigFromManifest(...)` - already calls installDomain
- `readBuiltInSteamInputTemplate(...)` - reads from assets, could stay
- `readDownloadedSteamInputTemplate(...)` - file read, could stay
- `resolveSteamControllerVdfText(...)` - orchestration that uses above methods

The first two should be removed, last three can stay as simple helpers or move to inputManager.

## Scope

- In scope:
  - Remove redundant wrapper methods that just call installDomain
- Out of scope:
  - Changes to SteamInputManager implementation

- [ ] Redundant wrappers removed from companion
