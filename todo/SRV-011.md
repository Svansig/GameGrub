# SRV-011 - Move Windows launch info helper to SteamInstallDomain

- **ID**: `SRV-011`
- **Area**: `service/steam`
- **Priority**: `P3`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

`getWindowsLaunchInfos(appId)` is a simple filter on app config that could stay in companion as simple helper or move to libraryDomain. It filters launch entries for .exe files.

Also, the `FileChanges` data class in companion is a simple data holder that could move to session domain or cloud stats domain.

## Scope

- In scope:
  - Either keep as simple helper or move to domain
  - Move FileChanges to appropriate domain
- Out of scope:
  - Any changes to launch logic

## Acceptance Criteria

- [ ] FileChanges moved to appropriate domain

- [ ] Build passes
