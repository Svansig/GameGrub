# SRV-012 - Remove cloud stats wrapper facades from companion

- **ID**: `SRV-012`
- **Area**: `service/steam`
- **Priority**: `P2`
- **Status**: `Backlog` (Low priority - requires extensive call site migration)
- **Owner**: `TBD`

## Problem

Cloud stats wrapper methods in SteamService companion just delegate to cloudStatsDomain:
- `cachedAchievements`, `cachedAchievementsAppId` - already in domain as StateFlow
- `generateAchievements(...)` - delegates to domain
- `clearCachedAchievements()` - delegates to domain
- `getGseSaveDirs(...)` - delegates to domain
- `syncAchievementsFromGoldberg(...)` - delegates to domain
- `storeAchievementUnlocks(...)` - delegates to domain

After accountDomain migration, these wrappers are redundant.

## Scope

- In scope:
  - Remove wrapper methods
  - Update call sites to use domain directly
- Out of scope:
  - Any changes to cloud stats domain implementation

## Acceptance Criteria

- [ ] Wrapper methods removed from companion
- [ ] Call sites updated
