# SRV-014 - Remove DAO injections from SteamService

- **ID**: `SRV-014`
- **Area**: `service/steam`
- **Priority**: `P1`
- **Status**: `Done`
- **Owner**: `Sisyphus`

## Problem

SteamService still has direct DAO injections that should be removed:
- `db: GameGrubDatabase` - should use domain transaction boundaries
- `licenseDao: SteamLicenseDao` - should go through libraryDomain
- `appDao: SteamAppDao` - should go through libraryDomain

The ownership matrix says these should be moved to domain transaction boundaries.

## Scope

- In scope:
  - Remove direct DAO injections from SteamService
  - Ensure domains are used for all database operations
- Out of scope:
  - Changes to DAO implementations

## Acceptance Criteria

- [ ] No direct DAO injections in SteamService
- [ ] All DB operations go through domains

- [ ] Build passes
