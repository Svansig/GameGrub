# SRV-015 - Consolidate companion constants into domain config

- **ID**: `SRV-015`
- **Area**: `service/steam`
- **Priority**: `P3`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

SteamService companion has constants that should be moved to appropriate domains:
- `MAX_PICS_BUFFER` - belongs in SteamPicsSyncDomain config
- `MAX_RETRY_ATTEMPTS` - belongs in connection policy
- `INVALID_APP_ID`, `INVALID_PKG_ID` - belong in catalog/library constants
- `STEAM_CONTROLLER_CONFIG_FILENAME` - belongs in SteamInstallDomain
- `requestTimeout`, `responseTimeout` - belong in connection config

These are configuration values that belong with their respective domain owners.

## Scope

- In scope:
  - Move constants to appropriate domain objects
  - Remove from companion
- Out of scope:
  - Changes to how constants are used
