## Problem

`SteamWorkshop` handler is currently removed during Steam client setup. We need to determine whether workshop integration would deliver meaningful value for Android users and whether implementation risk is acceptable.

## Scope

- In scope:
  - Assess user value for workshop-enabled titles in GameNative.
  - Evaluate feasibility for subscribe/download/update workflows in current architecture.
  - Assess storage and lifecycle impacts (foreground service, sync, cleanup).
  - Produce a `Go / Defer / Drop` decision with rationale.
- Out of scope:
  - End-to-end workshop UI implementation.

## Acceptance Criteria

- [ ] Usefulness assessment includes title examples and expected usage frequency.
- [ ] Feasibility assessment includes handler lifecycle and error-handling plan.
- [ ] Risk assessment includes storage growth, corruption, and support costs.
- [ ] Decision recorded as `Go`, `Defer`, or `Drop`.
- [ ] Documentation updated (or `No doc changes required` note added with reason).
