## Problem

`SteamScreenshots` handler is currently removed at startup. We need to determine whether screenshot capture/sync support should be part of GameNative priorities for Steam-on-Android use.

## Scope

- In scope:
  - Assess user value and usage frequency vs implementation cost.
  - Evaluate feasibility for screenshot upload/sync in Android + container setup.
  - Identify privacy/storage implications and moderation/support concerns.
  - Produce a `Go / Defer / Drop` decision with rationale.
- Out of scope:
  - Full screenshot gallery/editor implementation.

## Acceptance Criteria

- [ ] Usefulness assessment includes at least one high-value user flow.
- [ ] Feasibility assessment includes storage, background execution, and failure handling.
- [ ] Risk assessment includes privacy and support implications.
- [ ] Decision recorded as `Go`, `Defer`, or `Drop`.
- [ ] Documentation updated (or `No doc changes required` note added with reason).
