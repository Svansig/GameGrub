# SRV-037 - Migrate service-layer marker/file operations to storage gateways

## Problem

Service-layer file and marker operations are mixed with service orchestration logic and bypass a unified storage boundary.

## Scope

- In scope:
  - Move service-layer file IO and marker operations to `StorageFileGateway`/`StorageMarkerGateway`.
  - Remove direct marker/file helper dependencies from migrated service paths.
  - Keep behavior parity for install/download markers and recovery behavior.
- Out of scope:
  - Non-storage service domain refactors.
