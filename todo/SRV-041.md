# SRV-041 - Deduplicate MIME and encoding protocol literals in service/UI owners

- **ID**: `SRV-041`
- **Area**: `service/* + ui + api`
- **Priority**: `P2`
- **Status**: `Backlog`
- **Owner**: `TBD`

- **Reviewer**: `TBD`

## Problem

Protocol literals like `application/json`, `text/plain`, and `UTF-8` are repeated across service and UI code, causing avoidable duplication and inconsistency risk.

## Scope

- In scope:
  - Replace repeated protocol strings with owner-level constants.
  - Reuse existing constants where already available.
  - Keep these values out of Android string resources.
- Out of scope:
  - API contract changes.
