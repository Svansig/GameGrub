# UI-001 - Move GOG app operations out of UI class

- **ID**: `UI-001`
- **Area**: `ui/screen/library/appscreen`
- **Priority**: `P1`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

`GOGAppScreen` directly performs download/install/uninstall orchestration through service calls. This blends rendering-facing code with business operations.

## Scope

- In scope:
  - Route install/download/uninstall actions through ViewModel/domain-facing interfaces.
  - Keep UI file focused on action dispatch and state rendering.
- Out of scope:
  - Redesign of underlying GOG service internals.

## Acceptance Criteria

- [ ] No direct operation orchestration from `GOGAppScreen` for install/download/uninstall.
- [ ] UI actions are represented as intents/events to a ViewModel or equivalent coordinator.
- [ ] Existing user flows behave the same.

## Validation

- [ ] `./gradlew testDebugUnitTest`
- [ ] `./gradlew lintKotlin`
- [ ] Manual checks for install, cancel, uninstall paths.
