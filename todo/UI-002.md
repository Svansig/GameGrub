# UI-002 - Remove unmanaged IO scopes from GOG UI paths

- **ID**: `UI-002`
- **Area**: `ui/screen/library/appscreen`
- **Priority**: `P1`
- **Status**: `In Progress`
- **Owner**: `Sisyphus`

## Problem

`GOGAppScreen` uses `CoroutineScope(Dispatchers.IO).launch` in UI paths. Those jobs are not lifecycle-owned by a ViewModel and can outlive screen state.

## Scope

- In scope:
  - Replace ad-hoc IO scopes with lifecycle-aware ViewModel scope handling.
  - Ensure cancellation behavior is deterministic.
- Out of scope:
  - Service-level coroutine architecture changes.

## Acceptance Criteria

- [ ] No `CoroutineScope(Dispatchers.IO)` usage remains in target GOG UI file.
- [ ] Work is triggered via ViewModel actions and emits UI state/effects.
- [ ] Cancellation behavior on navigation away is validated.

## Validation

- [ ] `./gradlew testDebugUnitTest`
- [ ] `./gradlew lintKotlin`
- [ ] Manual navigation-away and action retry checks.
