# UI-003 - Remove unmanaged IO scopes from Steam UI paths

- **ID**: `UI-003`
- **Area**: `ui/screen/library/appscreen`
- **Priority**: `P1`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

`SteamAppScreen` uses unmanaged background scopes for update/download actions. This creates lifecycle and observability risk.

## Scope

- In scope:
  - Route side effects through ViewModel-managed scope.
  - Keep UI file as event dispatcher and state consumer.
- Out of scope:
  - Behavior changes to Steam download semantics.

## Acceptance Criteria

- [ ] Target UI file no longer starts unmanaged IO scopes.
- [ ] Steam action flow remains functionally equivalent.
- [ ] State/effect updates remain consistent with existing UX.

## Validation

- [ ] `./gradlew testDebugUnitTest`
- [ ] `./gradlew lintKotlin`
- [ ] Manual checks for download, cancel/resume, update actions.
