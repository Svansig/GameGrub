# UI-004 - Move library auth workflow out of composables

- **ID**: `UI-004`
- **Area**: `ui/screen/library`
- **Priority**: `P1`
- **Status**: `Backlog`
- **Owner**: `TBD`

## Problem

`LibraryScreen` directly owns OAuth launcher callback handling and credential-state checks through service/handler calls.

## Scope

- In scope:
  - Move auth processing and login-state derivation into ViewModel/state.
  - Keep composables focused on UI event hooks and rendering.
- Out of scope:
  - Replacing OAuth Activities.

## Acceptance Criteria

- [ ] OAuth result processing is handled by ViewModel/domain-facing layer.
- [ ] UI screen does not directly compute source login status via service singletons.
- [ ] Existing login success/error UX remains unchanged.

## Validation

- [ ] `./gradlew testDebugUnitTest`
- [ ] `./gradlew lintKotlin`
- [ ] Manual checks for GOG/Epic/Amazon login and cancellation paths.
