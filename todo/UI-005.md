# UI-005 - Decompose orchestration from GameGrubMain

- **ID**: `UI-005`
- **Area**: `ui/GameGrubMain.kt`
- **Priority**: `P1`
- **Status**: `In Progress`
- **Owner**: `Sisyphus`

## Problem

`GameGrubMain` currently combines routing, UI composition, and large orchestration workflows, reducing readability and testability.

## Scope

- In scope:
  - Extract launch/session orchestration into coordinator/use-case classes.
  - Preserve current navigation and user-visible behavior.
- Out of scope:
  - Full service redesign.

## Decomposition Plan (Current Pass)

Target file: `app/src/main/java/app/gamegrub/ui/GameGrubMain.kt`

1. Extract launch/session orchestration side effects into coordinator/use-case classes under `ui/model` or `launch` ownership.
2. Replace direct global reads/writes (`PrefManager`, service companion reads) with gateway-backed or ViewModel-provided state.
3. Keep `GameGrubMain` focused on route rendering, state collection, and event bridging only.

### Implementation Checklist

- [ ] Identify and isolate non-UI orchestration blocks (launch request handling, auth/session orchestration, service start/stop orchestration).
- [ ] Create testable coordinator/use-case units with explicit input/output contracts.
- [ ] Migrate call sites in `GameGrubMain` to new boundaries with behavior parity.
- [ ] Add or update tests for extracted orchestration units and regression checks for launch-entry routes.
- [ ] Remove now-dead helper blocks from `GameGrubMain`.

## Acceptance Criteria

- [ ] Main composable remains focused on navigation/rendering/event bridging.
- [ ] Extracted orchestration units are testable in isolation.
- [ ] No regressions in launch path handling.

## Validation

- [ ] `./gradlew testDebugUnitTest`
- [ ] `./gradlew lintKotlin`
- [ ] Manual checks for launch from UI and external intent entry.
