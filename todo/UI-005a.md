## Problem

`preLaunchApp` function (lines 1465-2026) is a 560+ line monolith in GameGrubMain that handles game launch orchestration. This makes it hard to test and reason about.

## Scope

- In scope:
  - Extract `preLaunchApp` function into a `LaunchCoordinator` class/object
  - Use Hilt injection for the coordinator
  - Ensure all existing parameters and callbacks work the same
  - Move unmanaged `CoroutineScope(Dispatchers.IO).launch` to lifecycle-managed scope
- Out of scope:
  - Changing any launch logic behavior

## Acceptance Criteria

- [x] Launch orchestration extracted into dedicated launch module (`ui/launch`).
- [x] `GameGrubMain` now delegates to extracted launch orchestration.
- [x] Unmanaged `CoroutineScope(Dispatchers.IO)` in `GameGrubMain` launch flow replaced with caller-owned lifecycle scope.
- [x] Existing launch parameters/behavior contracts preserved for call sites.
- [x] Manual behavior parity pass across all launch scenarios.

## Changes Made

- Extracted `preLaunchApp` to `ui/launch/GameLaunchOrchestrator.kt`
- Function now accepts `scope: CoroutineScope` parameter for lifecycle management
- Call sites in `GameGrubMain` pass `viewModel.viewModelScope` instead of creating unmanaged IO scopes
- All parameters and callback contracts preserved
