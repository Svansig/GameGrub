# UI-005c - Move launch resolution and Steam login gating out of UI launch orchestrator

- **Priority**: `P1`
- **Status**: `Reopened`

## Problem

`GameLaunchOrchestrator` owned install-state resolution and Steam login policy checks that are launch-domain concerns, not UI-orchestration concerns.

## Scope

- In scope:
  - Move `GameResolutionResult`, `resolveGameAppId`, and `needsSteamLogin` into a launch-domain boundary.
  - Update `GameGrubMain` imports/callers to use the new boundary.
- Out of scope:
  - Service API redesign.
  - Launch flow behavior changes.

## Acceptance Criteria

- [x] Resolution model/function moved out of `ui/launch`.
- [x] Steam login gate policy moved out of `ui/launch`.
- [x] `GameGrubMain` compiles against moved ownership surface.

## Changes Made

- Moved `GameResolutionResult`, `resolveGameAppId` to `app/gamegrub/launch/GameLaunchResolution.kt`
- Moved `needsSteamLogin` to launch-domain boundary
- Updated `GameGrubMain` imports to use new `app.gamegrub.launch` package
