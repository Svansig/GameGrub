# UI-005e - Isolate launch-request UI messaging helpers from launch orchestration core

## Problem

`GameLaunchOrchestrator` mixed launch pipeline logic with UI message helpers (`consumePendingLaunchWithError`, `showGameNotInstalledDialog`), making ownership blurred.

## Scope

- In scope:
  - Move UI-only launch request messaging helpers into dedicated UI helper file.
  - Keep behavior and dialog copy unchanged.
- Out of scope:
  - Dialog design changes.
  - Snackbar infrastructure changes.

## Acceptance Criteria

- [x] UI messaging helpers are extracted from launch orchestration core file.
- [x] Existing call sites keep behavior parity.
- [x] `GameLaunchOrchestrator` remains focused on launch prep/dispatch orchestration.

## Changes Made

- Extracted `consumePendingLaunchWithError` and `showGameNotInstalledDialog` to `ui/launch/LaunchRequestUiHandlers.kt`
- These helpers now handle UI-only concerns (snackbar, dialog state) while launch orchestration handles launch preparation logic
