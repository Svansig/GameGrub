# UI-005f - Extract service startup lifecycle policy from GameGrubMain

## Problem

`GameGrubMain` currently decides when to start Steam/GOG/Epic/Amazon services inside `repeatOnLifecycle`, which couples composable lifecycle to service policy logic.

## Scope

- In scope:
  - Move service startup/reconnect policy evaluation out of `GameGrubMain` into a coordinator/use-case boundary.
  - Keep existing startup conditions and route behavior unchanged.
  - Keep `GameGrubMain` as a caller that forwards lifecycle events/intents.
- Out of scope:
  - Service implementation redesign.
  - Login/auth protocol changes.

## Acceptance Criteria

- [x] Service startup decision logic no longer resides directly in `GameGrubMain`.
- [x] `GameGrubMain` delegates startup orchestration to a dedicated boundary.
- [x] Existing reconnect/start behavior parity is preserved for Steam/GOG/Epic/Amazon.

## Changes Made

- Created `ui/service/ServiceStartupCoordinator.kt` to handle service startup/reconnect policy
- `GameGrubMain` now calls `ServiceStartupCoordinator` instead of directly managing service lifecycle
- Coordinator handles Steam/GOG/Epic/Amazon service start conditions and reconnection logic
