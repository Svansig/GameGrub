## Problem

`GameGrubMain` currently performs update check, download, and install orchestration directly in composable effects and dialog callbacks.

## Scope

- In scope:
  - Move update check and install workflow orchestration behind ViewModel/coordinator boundary.
  - Keep dialog copy, loading behavior, and failure messaging unchanged.
- Out of scope:
  - Update backend/API changes.
  - Installer implementation changes.

## Acceptance Criteria

- [x] Update check/install orchestration no longer executes directly in composable body callbacks.
- [x] UI uses ViewModel/coordinator intents/effects for update flow.
- [x] Existing update dialog and error behavior remains parity.

## Changes Made

- Created `ui/update/AppUpdateCoordinator.kt` to handle update check, download, and install workflow
- `GameGrubMain` now uses `AppUpdateCoordinator` instead of direct update orchestration
- Coordinator handles update dialog state, download progress, and install failure handling
