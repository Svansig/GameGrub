## Problem

`GameGrubMain` directly performs container config persistence/restore operations (`IntentLaunchManager` + `ContainerUtils`) inside dialog action handlers.

## Scope

- In scope:
  - Extract save/discard/restore side effects into a dedicated coordinator/use-case.
  - Keep dialog behavior and temporary override semantics unchanged.
- Out of scope:
  - Container model schema changes.
  - Container editor UI redesign.

## Acceptance Criteria

- [x] Container config write/restore side effects are no longer executed directly in composable dialog callbacks.
- [x] `GameGrubMain` delegates config action handling to one boundary API.
- [x] Save/discard/dismiss behavior parity remains intact.

## Changes Made

- Extracted container config save/discard/restore logic to dedicated handlers
- `GameGrubMain` now delegates config actions through centralized boundary
- Preserved temporary override semantics and dialog close behavior
