# UI-005i - Move game feedback submission side effects out of GameGrubMain

- **ID**: `UI-005i`
- **Area**: `ui/GameGrubMain.kt + feedback flow`
- **Priority**: `P1`
- **Status**: `Done`
- **Owner**: `Sisyphus`
- **Parent**: `todo/UI-005.md`
- **Documentation Impact**: `Ticket docs updated to track feedback-flow ownership migration.`

## Problem

`GameGrubMain` currently owns feedback submission orchestration (worker call, try/catch policy, and result snackbar mapping), reducing testability of feedback behavior.

## Scope

- In scope:
  - Move feedback submission side effects behind ViewModel/use-case/coordinator boundary.
  - Keep visible user behavior (success/failure messaging, dialog close behavior) unchanged.
- Out of scope:
  - Feedback payload schema changes.
  - Backend endpoint changes.

## Acceptance Criteria

- [x] Feedback submission logic no longer runs directly in composable callback code.
- [x] UI callback forwards intent and renders resulting UI state/effects.
- [x] Existing success/failure snackbar behavior and dialog closing parity is preserved.

## Changes Made

- Moved feedback submission orchestration to `ui/feedback/GameFeedbackCoordinator.kt`
- `GameGrubMain` now forwards feedback intents through the coordinator
- Success/failure snackbar mapping handled by coordinator
