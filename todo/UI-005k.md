# UI-005k - Extract message-dialog action mapping from GameGrubMain monolith

## Problem

`GameGrubMain` contains a large `when (msgDialogState.type)` block that mixes UI mapping with side-effectful launch/session/config actions.

## Scope

- In scope:
  - Extract dialog action mapping into dedicated mapper/coordinator units.
  - Keep `MessageDialog` rendering in `GameGrubMain`.
  - Keep all button semantics and side effects behavior-compatible.
- Out of scope:
  - Dialog visual redesign.
  - Dialog type contract redesign.

## Acceptance Criteria

- [x] Dialog action mapping no longer lives as a monolithic branch in `GameGrubMain`.
- [x] `GameGrubMain` delegates dialog behavior resolution to dedicated helpers/coordinator.
- [x] Existing per-dialog confirm/dismiss/action behavior remains parity.

## Changes Made

- Created `ui/dialog/MessageDialogCoordinator.kt` to handle dialog action mapping
- Extracted dialog callbacks (onConfirmClick, onDismissClick, onActionClick) to coordinator
- `GameGrubMain` now uses coordinator to resolve dialog behaviors rather than large when block
