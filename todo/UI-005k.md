## Problem

`GameGrubMain` contains a large `when (msgDialogState.type)` block that mixes UI mapping with side-effectful launch/session/config actions.

## Scope

- In scope:
  - Extract dialog action mapping into dedicated mapper/coordinator units.
  - Keep `MessageDialog` rendering in `GameGrubMain`.
  - Keep all button semantics and side effects behavior-compatible.
- Out of scope:
  - Dialog visual redesign.
  - Dialog type contract redesign.

## Acceptance Criteria

- [ ] Dialog action mapping no longer lives as a monolithic branch in `GameGrubMain`.
- [ ] `GameGrubMain` delegates dialog behavior resolution to dedicated helpers/coordinator.
- [ ] Existing per-dialog confirm/dismiss/action behavior remains parity.
