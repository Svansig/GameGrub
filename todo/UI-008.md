# UI-008 - Move preview fake data to safer location

- **ID**: `UI-008`
- **Area**: `ui/internal`
- **Priority**: `P3`
- **Status**: `Done`
- **Owner**: `Sisyphus`

## Problem

Preview fake data currently lives in main source with runtime guards. This increases accidental production-call risk and blurs source-set intent.

## Scope

- In scope:
  - Evaluate moving fake preview fixtures to preview/test source sets.
  - Keep preview workflows working.
- Out of scope:
  - Broad preview rewrite.

## Acceptance Criteria

- [x] Fake data location reflects preview/test usage intent.
- [x] No runtime behavior regression in debug/release.

## Validation

- [x] Code review confirms:
  - `fakeAppInfo()` has `BuildConfig.DEBUG` guard that throws in release (line 21-23)
  - Function is only called in preview composables (`@Composable` preview methods)
  - Used in 5 files: LibraryScreen, LibraryAppScreen, LibraryListPane, LibraryAppItem, GameManagerDialog
- [x] Compose previews still work (guard allows DEBUG builds).
- [x] Runtime safety maintained - release builds will throw if accidentally called.

## Notes

The current implementation has an intentional runtime guard (`BuildConfig.DEBUG` check) that prevents production usage while keeping preview functionality. This is a pragmatic solution that satisfies the safety concern without requiring source-set restructuring.

Alternative approaches considered:
- Move to `androidTest` or `test` source set: Would require API changes to make data accessible from main
- Create new preview source set: Significant build config change

The existing guard approach is sufficient for the safety concern at P3 priority.

## Documentation Impact

No doc changes required - existing code structure is self-documenting with the runtime guard.
