# UI-009 - Define legacy UI seam guardrails

- **ID**: `UI-009`
- **Area**: `com.winlator + res/layout`
- **Priority**: `P3`
- **Status**: `Done`
- **Owner**: `Sisyphus`

## Problem

The app has a Compose-first UI and a legacy XML/View seam in `com.winlator`. Without explicit guardrails, responsibilities can drift into the legacy stack.

## Scope

- In scope:
  - Document and enforce exception boundary.
  - Prevent new app-level UI feature logic from landing in legacy seam by default.
- Out of scope:
  - Full migration of Winlator UI components.

## Acceptance Criteria

- [x] Exception boundary is clearly documented.
- [x] New PRs touching legacy seam reference justification when needed.
