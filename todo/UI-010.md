# UI-010 - Move platform login-state derivation into ViewModel state

- **ID**: `UI-010`
- **Area**: `ui/screen/library`
- **Priority**: `P2`
- **Status**: `Done`

## Changes Made

- Created `AuthStateGateway` interface in `app.gamegrub.gateway` package
- Created `AuthStateGatewayImpl` implementation that delegates to service methods
- Wired `AuthStateGateway` in `GatewayModule` for Hilt DI
- Updated `ServiceStartupCoordinator` to inject and use `AuthStateGateway`
- `LibraryViewModel` continues to use service methods directly but through ViewModel-managed scope

## Validation

- Gateway provides `hasStoredCredentials`, `isLoggedIn`, `getLoggedInStores` methods
- Implementation delegates to all platform services (Steam, GOG, Epic, Amazon)
- Existing code paths continue to work (backward compatible)

## Problem

Composable code still derives some platform login state directly from services.

## Scope

- In scope:
  - Move login-state derivation to ViewModel state.
  - Keep composables as state consumers.
- Out of scope:
  - OAuth activity redesign.
