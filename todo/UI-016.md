# UI-016 - Replace global launch/network reads in UI with injected state gateways

## Problem

`GameGrubMain` and related UI flows still read launch/network global state directly, increasing coupling and reducing testability.

## Scope

- In scope:
  - Route pending-launch and network-connected state into ViewModel via injected gateway interfaces.
  - Remove direct global reads from target UI code paths.
- Out of scope:
  - Replacing all app event systems.
