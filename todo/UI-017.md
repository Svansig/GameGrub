## Problem

Direct `PrefManager` reads in composables keep UI tied to global mutable state and complicate testing.

## Scope

- In scope:
  - Move selected composable preference reads to ViewModel-provided state.
  - Use gateway-backed preference access where needed.
- Out of scope:
  - Migrating all preference reads in one pass.

## Analysis

- Found 19 files in ui/screen using PrefManager directly
- Pattern varies: reads for settings, init calls, configuration access
- PreferencesGateway exists and is wired in DI
- Large-scale migration requires significant ViewModel refactoring
- COH-016 completed gateway setup - actual usage migration is separate work
- LibraryViewModel already handles many preference-driven state items
