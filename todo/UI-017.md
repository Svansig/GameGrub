# UI-017 - Remove direct PrefManager access from target composables via ViewModel state

## Problem

Direct `PrefManager` reads in composables keep UI tied to global mutable state and complicate testing.

## Scope

- In scope:
  - Move selected composable preference reads to ViewModel-provided state.
  - Use gateway-backed preference access where needed.
- Out of scope:
  - Migrating all preference reads in one pass.
