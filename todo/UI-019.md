## Problem

Orientation policy events are emitted from composable bodies, which can fire repeatedly on recomposition and create unnecessary churn and hard-to-reason behavior.

## Scope

- In scope:
  - Move orientation policy updates into `LaunchedEffect`/lifecycle-bound effect blocks.
  - Ensure route transitions emit orientation policy updates once per meaningful change.
  - Remove temporary orientation workaround comments once behavior is stable.
- Out of scope:
  - Refactoring unrelated UI event emissions.
  - Changing orientation feature scope.
