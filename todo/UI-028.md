## Problem

Hardcoded user-facing strings remain in key UI paths (main dialogs/snackbars, settings debug UX, feedback submission messaging), creating localization drift and inconsistent copy ownership.

## Scope

- In scope:
  - Move audited user-facing literals to `app/src/main/res/values/strings.xml`.
  - Replace call sites with `stringResource(...)` / `context.getString(...)`.
  - Preserve existing behavior, severity, and dynamic placeholders.
- Out of scope:
  - Copywriting redesign or product wording changes.
